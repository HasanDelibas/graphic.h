Tek Arabellekli hareket

Basit hareket ettirme metodu

Arkadaþlar nesneleri hareket ettirmenin bir animasyon hazýrlamanýn en basit yöntemi ekraný tamamýyla silmek(1), nesnelerin yeni konumunu belirlemek(2) ve nesneleri çizdirmektir(3).


Þimdi örnek kodlarla bu algoritmayý inceleyelim.

 

#include <graphics.h>

int yon=1;

int konumx=50;

void yenikonum(){

    if (konumx<351 && yon==1)  konumx+=5;

    if (konumx==350) yon=0;

    if (konumx>49 && yon==0) konumx-=5;

    if (konumx==50) yon=1;

    }

int main()

{ 

    initwindow(400,300, "4. Nesneleri Hareket Ettirme");

    while(1){

    cleardevice();

    yenikonum();

    setcolor(255);

    circle(konumx,50,20);

    setcolor(COLOR(0,255,0));

    circle(50,150,20);

    delay(40);            

    }    

    closegraph();    

    return 0;

}

 

yenikonum(); kýsaca x=50 ile x=350 arasýnda gidip gelen bir noktanýn yeni konumunu belirlemek için kullandýk.

Kodlardaki animasyon algoritmasý þu kodlardadýr.

 

    while(1){

    cleardevice();

    yenikonum();

    setcolor(255);

    circle(konumx,50,20);

    setcolor(COLOR(0,255,0));

    circle(50,150,20);

    delay(40);            

    }    

 

Döngü içerisindeki kodlarý açýklayalým.

 

cleardevice ( );

cleardevice();  Ekraný arkaplan rengine göre tamamen temizler.

 

circle ( x, y , r [yarýçap] );

circle(x,y,r); Merkezi x,y noktasý olan yarýçapý r olan bir daire çizer.

 

delay ( süre [milisaniye] ) ;

delay(s); s  milisaniye kadar programý bekletir. delay(1000); 1000 milisaniye = 1 saniye  kadar programý bekletir.

 

Filmler olsun oyunlar olsun genelde saniyede 25 kare fotoðrafýn ardý ardýna gelmesiyle gözümüzde canlýlýk kazanýr.

Bu da yaklaþýk her kareden sonra 1000 ms / 25 kare = 40 ms/kare beklemek gerekir. Bu iþlemide delay(40); ile yapýyoruz.

 

 

            Görüldüðü üzere önce ekraný temizledik, animasyon konumunu belirledik, nesneyi çizdirdik, ve sonraki animasyon için biraz bekledik.

 

 


Tek Arabellekli Hýzlý Animasyon

Tek arabellekli Basit Animasyonda görüntünün tamamýný siliyorduk. Fakat þimdi ekranýn tamamýný deðilde animasyonla deðiþicek yerleri  saptayýp eski yerleri temizleyip(1), animasyondaki  yeni yerleri saptayýp(2), Nesneleri çizdiriyoruz(3).

 

Örnek

 

#include <graphics.h>

int yon=1;

int konumx=50;

void yenikonum(){

    if (konumx<351 && yon==1)  konumx+=5;

    if (konumx==350) yon=0;

    if (konumx>49 && yon==0) konumx-=5;

    if (konumx==50) yon=1;

    }

int main()

{ 

    initwindow(400,300, "4. Nesneleri Hareket Ettirme");

    yenikonum();

    setcolor(255);

    circle(konumx,50,20);

    setcolor(COLOR(0,255,0));

    circle(50,150,20);

    delay(40);            

    }

Bir önceki örneðin aynýsý fakat deðiþen kodlar þöyle;

    yenikonum();

cleardevice(); yi kaldýrdýk. Yerine eski noklarý silmek için  setcolor(0); ve circle(konumx,50,20); bu sayede ekranýn tamamýný temizlemek yerine sadece animasyon yapýlan bölgeyi temizledik.(1)

 

Ardýndan yenikonum(); ile animasyonumuzdaki yeni konumlarý belirledik.(2)

 

Son olarak da setcolor(255); ve circle(konumx,50,20) ile yeni animasyonumuzu çizdirdik.(3)

 

delay(40);  bizim animasyonda olmazsa olmazýmýzdý zaten.

Çift Arabellekli Animasyon

 

void swapbuffers ( );

 

Çift Arabellekli Animasyon oyun sanayisinin olmazsa olmazýdýr. Günümüzdeki oyunlarýn bir çoðu bu teknikle animasyon yapar. Çift Arabellekli Animasyonu anlatmak gerekirse; çizim iþlemlerini kullanýcýya farkettirmeden arka bellekte yapar görüntülemeyi ön bellekte yapar. Çizim iþlemleri bittikten sonra yeni kare arka bellekten, ön belleðe swapbuffers(); kodu ile geçiþ yapar. Bu sayede kullanýcý her zaman çizimleri tamamen bitmiþ görüntüleri güçlü bir þekilde gözü yorulmadan görür.

 

Örnek :

 

#include <graphics.h>

int yon=1;

int konumx=50;

void yenikonum(){

    if (konumx<351 && yon==1)  konumx+=5;

    if (konumx==350) yon=0;

    if (konumx>49 && yon==0) konumx-=5;

    if (konumx==50) yon=1;

    }

int main()

{ 

    initwindow(400,300, "4. Nesneleri Çift Arabellekli Sistem ile Hareket Ettirme");  

    swapbuffers();

    while(1){

    cleardevice();

    yenikonum();

    setcolor(255);

    circle(konumx,50,20);

    setcolor(COLOR(0,255,0));

    circle(50,150,20);

    swapbuffers();

    delay(40);           

    }    

    closegraph();    

    return 0;

}

 

Ek olarak sadece swapbuffers(); ’ i ekledik. Daha öncesinde dediðimiz gibi arka bellekteki görüntüyü ön bellek ile deðiþtirir.

 

 swapbuffers(); ’ý tek kullanýmlýk olarak initwindow(); fonksiyonundan sonra kullanmanýzý öneririz yada initwindow() ‘ un dbflag parametresini true yapmanýzý öneririz. Çünkü ilk çizimdelerde uygulama Çift Arabellekli Animasyon Kullandýðýmýzý bilmiyecektir ve bu yüzden ilk çizim animasyonunu Tek Arabellekli Animasyon olmasýný istemediðimiz için eklemeliyiz.

 

swapbuffers(); fonksiyonunu bir kez dahi olsa kullandýðýnýz anda program Çift Arabellekli Çizim Metoduna geçtiði için görüntünün yeninlenmesini istediðiniz zamanlarda swapbuffers(); kullanmanýz zorunludur. Yoksa uygulamanýzda yeni görüntüler oluþmaz.
